# Resetting State

> **TL;DR**: Each generated test case assumes the system starts in a known initial state. Use `BeforeEachAsync` in `TestExecutionOptions` to reset your system before each test case runs.

---

## Why State Reset Matters

When Accordant generates test cases, each one starts from the **initial state** you provide. If the system under test doesn't actually correspond to that initial state, tests will fail unpredictably—not because of bugs, but because of leftover state from previous runs.

```csharp
var initialState = new BankState();
var testCases = spec.GenerateTests(initialState, inputs, options);
```

Every test case expects the real system to match `initialState` before execution begins.

---

## Using BeforeEachAsync

Accordant provides a `BeforeEachAsync` callback in `TestExecutionOptions` that runs before each test case. This is where you reset your system:

```csharp
var results = await spec.RunTests(context, initialState, testCases, new TestExecutionOptions
{
    BeforeEachAsync = async _ =>
    {
        // Reset system to initial state here
        foreach (var accountId in knownAccounts)
        {
            await client.DeleteAccount(accountId);
        }
    }
});
```

---

## Reset Strategies

The right approach depends on your system, but at a high level you can:

- **Recreate the resource** — tear down and rebuild the system from scratch
- **Delete resources the tests could have created** — clean up known entities before each test
- **Isolate with unique names** — give each test run its own namespace

### 1. Recreate the Resource

If your system is cheap to create (in-memory services, test containers), recreate it before each test:

```csharp
var results = await spec.RunTests(context, initialState, testCases, new TestExecutionOptions
{
    BeforeEachAsync = async _ =>
    {
        // Recreate the service with fresh state
        await testContainer.ResetAsync();
    }
});
```

This gives perfect isolation but may be slow for heavier resources.

### 2. Delete Resources the Tests Could Have Created

If your service exposes delete endpoints, clean up known entities in `BeforeEachAsync`:

```csharp
// Known entities used in tests
var knownAccounts = new[] { "alice", "bob" };

var results = await spec.RunTests(context, initialState, testCases, new TestExecutionOptions
{
    BeforeEachAsync = async _ =>
    {
        // Clean up known accounts (ignore 404 if they don't exist)
        foreach (var accountId in knownAccounts)
        {
            await client.DeleteAccount(accountId);
        }
    }
});
```

This is the pattern used in the BankAccount sample.

### 3. Isolate with Unique Names

For services you can't easily reset (Azure Storage accounts, shared databases), isolate each test run using unique identifiers:

```csharp
var testRunId = Guid.NewGuid().ToString();

var results = await spec.RunTests(context, initialState, testCases, new TestExecutionOptions
{
    BeforeEachAsync = async testCase =>
    {
        // Each test case gets a unique container
        var containerName = $"test-{testRunId}-{testCase.Id}";
        await storageClient.CreateContainerAsync(containerName);
        context.Register(new ContainerContext(containerName));
    }
});
```

> **Note**: If your requests contain the resource name (e.g., account ID, container name), you may also need to transform the requests to use the unique name.

Choose the approach that works best for your system.

## Full Example: BankAccount

From the BankAccount sample—note how `BeforeEachAsync` deletes known accounts before each test:

```csharp
[Test]
public async Task AutoGeneratedSequentialTests()
{
    var spec = CreateSpec();
    
    // Bind operations to HTTP client...
    spec.ExecuteWith<BankApiClient>()
        .Bind<CreateAccountRequest, CreateAccountResponse>("CreateAccount",
            (client, req) => client.CreateAccount(req.AccountId).Result)
        .Bind<DeleteAccountRequest, DeleteAccountResponse>("DeleteAccount",
            (client, req) => client.DeleteAccount(req.AccountId).Result);
    // ... other bindings

    // Generate test cases from initial state
    var initialState = new BankState();
    var testCases = spec.GenerateTests(initialState, inputs, new TestGenerationOptions
    {
        MaxDepth = 4
    });

    // Set up execution context
    var context = spec.CreateTestingContext();
    var httpClient = _factory.CreateClient();
    var client = new BankApiClient(httpClient);
    context.Register(client);
    
    // Known accounts used in tests
    var knownAccounts = new[] { "alice", "bob" };
    
    // Run tests with reset callback
    var results = await spec.RunTests(context, initialState, testCases, new TestExecutionOptions
    {
        BeforeEachAsync = async _ =>
        {
            // Clean up known accounts before each test (ignore 404 if they don't exist)
            foreach (var accountId in knownAccounts)
            {
                await client.DeleteAccount(accountId);
            }
        }
    });

    Assert.That(results.All(r => r.Success), Is.True);
}
```
